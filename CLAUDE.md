## Проект

**Amicus** (изначально шёл под рабочим названием «Круг» — переименован после старта разработки, см. project-brief.md) — мобильное приложение-соцсеть, где лента пользователя показывает посты только тех, с кем есть подтверждённое «знакомство» (Connection): через одноразовую invite-ссылку или QR-код при личной встрече. «Комнаты» (Room) — приватные под-пространства с собственной изолированной лентой — часть общей идеи проекта, но реализация отложена за пределы MVP (см. future-development.md).

Проект пилотный/портфолио: автор — новичок в мобильной разработке, но с опытом SQL/PostgreSQL и C#/.NET.

## Статус

Этапы 0–4 завершены и протестированы вручную на физическом устройстве (см. [docs/implementation-plan.md](docs/implementation-plan.md)). Есть экраны входа/регистрации, профиль с загрузкой аватара, таблица `users` (создаётся триггером `handle_new_user()`), экран «Знакомства» с invite-кодами и списком знакомых (`invite_links`/`connections` + RPC-функции), общая лента с публикацией постов (текст/фото), пагинацией, лайками и плоскими комментариями (`posts`/`reactions`/`comments`), CI зелёный. Подтверждение email включено (redirect после подтверждения ведёт на GitHub-репозиторий — deep link на сам экран приложения пока не настроен, см. future-development.md). Следующий шаг — Этап 5 (полировка + деплой).

Вне очереди (до Этапа 5, осознанное отступление от пошагового плана) добавлено:
- удаление своих постов и комментариев — DELETE RLS-политики на `posts`/`comments`/фото поста + подтверждение в UI, релиз [v0.2.0](https://github.com/tiltozavr2545/amicus/releases/tag/v0.2.0);
- несколько типов реакций (0.3.0) — вместо одного лайка три взаимоисключающие реакции 👍/😐/👎: колонка `type` в `reactions` (`like`/`neutral`/`dislike`, уникальность `(post_id, user_id)` осталась — одна реакция на пост) + UPDATE RLS-политика для смены реакции, upsert в репозитории, у каждой реакции свой счётчик, повторный тап снимает.
- приватность видимости данных (0.4.0) — сузили две широкие RLS-политики чтения, закрыв два пункта из раздела «Безопасность и приватность» future-development.md: (1) профили (`users`) теперь видны только себе и своим Connections вместо `using(true)` — раньше любой залогиненный мог перечислить всю адресную книгу; (2) реакции — SELECT только своих строк, а настоящий счётчик на пост отдаёт `security definer`-функция `reaction_summary()` (только числа + собственная реакция, без чужих `user_id`), так что «кто что поставил» больше не утекает, а счётчик остаётся истинным тоталом. Миграция `20260712100000_scope_profile_and_reaction_visibility.sql`.

См. future-development.md.

**Важно:** dev-машина на macOS 12.7.6 — версии Flutter новее 3.32.8 не запускаются (нужен macOS 14+). SDK закреплён на 3.32.8. iOS-разработка отложена (нужен Xcode, который тоже требует более новую macOS) — пока только Android.

## Стек

- **Frontend:** Flutter / Dart (кроссплатформенно, iOS + Android)
- **Backend:** Supabase (PostgreSQL + Auth + Storage + Realtime, Row Level Security для контроля доступа к постам)
- Альтернатива на будущее (не для 0.1): свой backend на ASP.NET Core + PostgreSQL

## Модель данных (черновик, дорабатывается по ходу)

Ключевые сущности MVP: `User`, `Connection` (user_a_id, user_b_id, method: invite_link/qr_code), `InviteLink`, `Post`, `Reaction`, `Comment` (с parent_comment_id для тредов, в MVP не используется). `Room`, `RoomMember`, `PostTarget` — вынесены в future-development.md вместе с реализацией комнат.

Полные поля — в разделе 3 брифа.
