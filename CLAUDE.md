## Проект

«Круг» (рабочее название, может смениться — см. бриф) — мобильное приложение-соцсеть, где лента пользователя показывает посты только тех, с кем есть подтверждённое «знакомство» (Connection): через одноразовую invite-ссылку или QR-код при личной встрече. «Комнаты» (Room) — приватные под-пространства с собственной изолированной лентой — часть общей идеи проекта, но реализация отложена за пределы MVP (см. future-development.md).

Проект пилотный/портфолио: автор — новичок в мобильной разработке, но с опытом SQL/PostgreSQL и C#/.NET.

## Статус

Этапы 0 и 1 завершены (см. [docs/implementation-plan.md](docs/implementation-plan.md)). Есть экраны входа/регистрации, профиль с загрузкой аватара, таблица `users` с RLS, CI зелёный. Следующий шаг — Этап 2 (Connections: invite-ссылки).

**Важно:** dev-машина на macOS 12.7.6 — версии Flutter новее 3.32.8 не запускаются (нужен macOS 14+). SDK закреплён на 3.32.8. iOS-разработка отложена (нужен Xcode, который тоже требует более новую macOS) — пока только Android.

## Стек

- **Frontend:** Flutter / Dart (кроссплатформенно, iOS + Android)
- **Backend:** Supabase (PostgreSQL + Auth + Storage + Realtime, Row Level Security для контроля доступа к постам)
- Альтернатива на будущее (не для v1): свой backend на ASP.NET Core + PostgreSQL

## Модель данных (черновик, дорабатывается по ходу)

Ключевые сущности MVP: `User`, `Connection` (user_a_id, user_b_id, method: invite_link/qr_code), `InviteLink`, `Post`, `Reaction`, `Comment` (с parent_comment_id для тредов, в MVP не используется). `Room`, `RoomMember`, `PostTarget` — вынесены в future-development.md вместе с реализацией комнат.

Полные поля — в разделе 3 брифа.

## Границы MVP (v1)

Делать: регистрация/вход, профиль, invite-ссылки → Connection, общая лента (текст + одно фото, видна всем Connections), лайк со счётчиком, **плоские** комментарии (без тредов).

Не делать в v1 (отложено, см. [future-development.md](docs/future-development.md)): комнаты (Rooms) целиком, несколько типов реакций, вложенные комментарии, QR-сканирование, push-уведомления, модерация/жалобы/блокировка, настройки приватности контактов, web-версия, свой backend взамен Supabase.

## Нерешённые вопросы (см. [future-development.md](docs/future-development.md))

Не блокируют работу, но всплывут по ходу: набор реакций, треды в комментариях, кто может добавлять участников в комнату, судьба комнаты при удалении создателя, разрыв Connection, приватность списка контактов. Решать по мере того, как доходит очередь до соответствующего этапа — не пытаться закрыть их заранее.
